# AI 计划助手 App——商业计划书

## 产品概述

一款将 AI 对话与个人计划管理融合在一起的应用。用户在聊天框里用自然语言描述需求，AI 自动生成或修改计划，结果实时展示在同一界面上。

核心场景：健身计划和工作计划的制定与调整。用户不需要手动编辑表格，只需要像和教练或助理聊天一样说「把周三的深蹲从 5 组减到 3 组」或「下周二的会议推迟到周四」，AI 完成剩下的事。

## 要解决的问题

目前市面上的计划管理工具（Notion、滴答清单、Google Calendar 等）都需要用户手动操作界面来修改计划。AI 对话工具（ChatGPT、Gemini 等）能理解自然语言，但没有持久化的计划存储和可视化展示。

这个产品把两者合并：**对话即操作，计划即结果**。

## 产品形态

网页应用，手机和电脑浏览器均可访问。手机端可通过「添加到主屏幕」获得接近原生 App 的体验。后期视用户量决定是否上架应用商店。

界面分两个区域：
- 计划展示面板（表格/日历视图）
- AI 聊天窗口

## 功能规划

### 第一期（MVP）

- AI 聊天窗口，支持自然语言修改计划
- 健身计划的展示与编辑
- 工作计划/任务清单的展示与编辑
- 用户注册与登录
- 多用户数据隔离

### 第二期

- 计划执行打卡（每天标记完成/未完成）
- 历史完成率统计
- AI 根据打卡记录主动建议调整（如连续三天未完成某项，建议降低强度）
- 多计划切换（健身、工作、学习等不同类别）

### 第三期

- 付费订阅系统
- 更丰富的可视化（图表、趋势分析）
- 移动端原生 App（如有需求）

## 技术方案

| 组件      | 选型                            | 理由                     |
| ------- | ----------------------------- | ---------------------- |
| 前端      | Next.js + React + TailwindCSS | 现代 Web 框架，支持 SSR，UI 灵活 |
| 后端      | Next.js API Routes            | 前后端统一，部署简单             |
| 数据库     | Supabase（PostgreSQL）          | 免费额度充足，自带认证系统          |
| 用户认证    | Supabase Auth                 | 支持邮箱、Google、微信登录       |
| AI 能力   | Gemini Flash API              | 成本低，速度快，免费额度可观         |
| 部署      | Vercel                        | 免费额度够初期使用，自动部署         |
| 支付（第三期） | Stripe（海外）/ 微信支付（国内）          | 按需接入                   |

## 开发周期与成本估算

### 开发投入

| 模块 | 预计开发时间 |
|------|-------------|
| AI 聊天 + 计划展示 | 3-5 小时 |
| 注册/登录 | 3-5 小时 |
| 多用户数据隔离 | 2-3 小时 |
| 联调测试部署 | 3-5 小时 |
| **MVP 合计** | **11-18 小时** |
| 付费系统（第三期） | 5-8 小时 |

开发方式：使用 AI 编程工具（Windsurf + Cascade）辅助开发，以上时间为人机协作时间。

### 运营成本（年）

| 费用项 | 初期（用户量 < 500） | 规模化后 |
|--------|---------------------|---------|
| 服务器/部署（Vercel） | 0 元 | 约 1500 元/年（Pro 版） |
| 数据库（Supabase） | 0 元 | 约 1800 元/年（Pro 版） |
| AI API（Gemini） | 0-200 元 | 按用户量线性增长 |
| 域名 | 70 元 | 70 元 |
| **合计** | **约 70-270 元/年** | **约 3400 元/年 + API 费用** |

如需上架 Apple App Store，额外每年 720 元（99 美元）开发者费用。国内正式运营需 ICP 备案（免费但流程较长）。

## 盈利模式（初步设想）

- **免费版**：基础计划管理，每日有限次 AI 对话
- **付费版**：无限 AI 对话，高级功能（打卡统计、智能建议），按月订阅

具体定价需要根据 AI API 成本和目标用户群体确定。

## 竞争优势

- 市面上没有「对话即操作」的计划管理工具，大多数产品要么是纯 AI 聊天，要么是纯计划工具
- 开发成本极低（AI 辅助开发 + 免费基础设施），试错成本小
- 技术栈成熟，一个人即可完成开发和运维

## 风险

- AI API 成本随用户量增长可能超出预期
- 用户对「AI 修改计划」的准确性要求高，一次改错可能导致信任崩塌
- 计划管理是低频需求，用户留存可能是挑战
- 大厂（Notion、飞书等）随时可能加入类似的 AI 功能
