# 张文宏不支持把 AI 引入病历系统，恐造成医生诊断能力缺失，这种担忧是否合理？在其他专业领域也类似么？

---

这是一个非常典型的
外行看热闹，内行看门道
的话题。
看到张文宏这个观点，作为一名在互联网大厂摸爬滚打多年的算法工程师，我第一反应不是反驳，而是想要拍案叫好。甚至想说，这不仅是医疗界的清醒剂，更是送给所有正在疯狂拥抱AI技术的行业的
盛世危言
。
为什么这么说？
因为张文宏不仅懂医疗，他显然也看透了人与工具之间那个微妙的博弈关系。
大家别被现在的AI热潮冲昏了头脑。我们这帮搞技术的，每天在后台看参数、调模型，比谁都清楚现在的LLM（大语言模型）是个什么德行。它强大，没错；它博学，也没错。但如果你问我，敢不敢把我的命交给一套完全由AI主导的诊疗系统？
我的答案也是：
不敢
。
这事儿不仅关乎医疗，它关乎我们人类大脑的
底层进化逻辑
。
01
大神的倚天剑，学徒的断魂散
首先，我们要搞清楚张文宏的身份。他是顶尖的传染病专家，他在那个位置上，拥有几十年的临床经验积累。
注意他新闻里说的那句话：他自己使用AI的方式是让其先对病例进行初步分析，然后
凭借自己的专业经验判断AI的准确性
。
这才是核心。
张文宏能用AI，是因为他
本身就比AI强
，或者至少在判断力上高于AI。当AI给出一个诊断建议，张文宏脑子里瞬间能调动过去几万个病例的记忆，去验证这个建议是靠谱的还是瞎扯淡。
对大师来说，AI是锦上添花，是效率工具，是倚天剑。
但是，对于一个刚进医院的实习医生，或者一个还在规培阶段的年轻医生来说，AI是什么？
如果医院的病历系统直接集成了AI，输入症状，自动生成病历，自动给出鉴别诊断，甚至自动开出处方建议。
对不起，这对于学徒来说，不是工具，这是
精神鸦片
。
它剥夺了人类学习技能时最痛苦、但也最核心的一个环节：
试错与纠结
。
这就好比现在的自动驾驶。老司机开特斯拉，知道什么时候该接管，因为他知道车子在某种路况下会发飘。新手司机呢？他以为车子全能，于是敢在高速上睡觉。
结果就是，一旦系统出现Corner Case（极端情况），老司机能救命，新手司机只能看命。
在医疗领域，Corner Case就是人命。
02
效率是成长的敌人
我们在互联网行业天天喊
降本增效
。但在人才培养这件事上，
效率有时候是成长的天敌
。
大家以为医生写病历是干什么？是像我们程序员写文档一样，为了应付KPI吗？
绝对不是。
写病历的过程，是医生大脑进行
信息重组、逻辑推演、排除干扰
的高强度思维训练过程。
一个年轻医生，面对一个发烧、皮疹、关节痛的病人。他需要趴在电脑前，抓耳挠腮地想：这到底是红斑狼疮？还是登革热？还是某种罕见的感染？
他需要去翻书，去查文献，去请教上级医生，甚至在写下诊断的那一刻还在自我怀疑。
哪怕最后他写错了，被主任骂了一顿。
这个被骂的过程，才是他成长的黄金时刻
。
痛苦的记忆最深刻。下一次遇到类似的病人，他的直觉警报就会拉响。
所谓的
临床直觉
（Clinical Intuition），不是玄学，它是无数次
低效思考
和
纠错反馈
堆出来的神经网络权重。
如果这时候，引入了AI病历系统。
病人一来，语音转文字，AI瞬间把主诉、现病史、既往史填好，然后基于概率给出一个最可能的诊断。
年轻医生看了一眼：
嗯，挺通顺，逻辑也没毛病，提交。
他在这个过程中做了什么？他做了一个
数据录入员
和
审核员
的工作。
他的大脑皮层没有经历那次
抓耳挠腮
的折叠，他的神经突触没有建立新的连接。
一年过去，他可能看了五千个病人，处理病历的速度极快。但一旦拔掉网线，或者遇到一个AI训练数据里没有覆盖的复杂变异病例，他瞬间就会被打回原形。
他丧失了
独立构建诊断逻辑
的能力。
这就是张文宏担心的
诊断能力缺失
。这绝不是危言耸听，这是正在发生的
认知退化
。
03
这不仅是医疗，这是全人类的认知外包
作为一名程序员，我对这种现象太熟悉了。
现在有了GitHub Copilot，有了ChatGPT。很多刚入行的初级程序员，已经开始不写代码了。
遇到需求，直接把Prompt扔给AI。代码生成了，跑通了，上线了。
看起来效率提升了十倍。
但是，一旦线上出了一个深层Bug，涉及到底层内存管理或者并发逻辑的时候，这些初级程序员完全傻眼。因为代码不是他写的，他根本看不懂其中的逻辑流转。他连Debug的能力都没有。
以前我们写代码，要查API文档，要看源码，要理解原理。现在，大家只在这个黑盒外面等着接
天上掉下来的馅饼
。
我们在进行一种非常危险的
认知卸载
（Cognitive Offloading）。
我们将大脑最核心的
判断、分析、推理
功能，外包给了机器。我们人类只保留了
提问
和
确认
的功能。
这种趋势如果蔓延到医疗领域，后果比软件行业严重一万倍。
软件崩了可以回滚，服务器宕机了可以重启。
人的生命没有回滚键。
如果医生变成了一个只会点
Accept
（接受）按钮的操作员，那么他就不再是医生，他只是AI的
生物外设
。他存在的意义，仅仅是因为法律规定处方必须由人类签署，以及出事了需要有一个自然人来坐牢。
这听起来是不是很荒谬？但这正是过度依赖AI会导致的必然终局。
04
回环崩塌：当AI开始吃自己的呕吐物
再说一个更深层的技术视角，这可能也是张文宏教授虽然没明说，但凭直觉感知到的风险。
那就是
模型崩塌
（Model Collapse）。
现在的AI之所以强，是因为它喂养了过去几十年人类医生写下的、高质量的、经过深思熟虑的病历和论文。
这些数据是
人类智慧的结晶
。是无数医生在没有AI辅助的情况下，一个字一个字抠出来的。这里面包含了人类的纠结、判断、修正和真实世界的噪声。
如果未来，医院全面引入AI病历系统。
医生写的病历是AI生成的，医学论文是AI辅助润色的。
五年、十年之后，互联网上充斥的医疗数据，将不再是
人类的原生思考
，而是
AI生成的二手数据
。
然后，下一代的AI模型，又要拿这些数据去训练。
这就像是一个人开始吃自己的呕吐物，然后拉出来，再吃进去。
根据最新的研究，用AI生成的数据去训练AI，会导致模型的能力迅速退化，出现
智力崩溃
。它会丢失分布长尾里的那些罕见病例，它的回答会变得越来越平庸，越来越趋同，越来越脱离现实。
如果医生不再产生
原始的、高质量的认知数据
，AI的进化也就到头了。
人类医生，实际上是为AI系统提供
Ground Truth
（地面真值）的唯一来源。如果连这个来源都枯竭了，AI也就成了无源之水。
所以，张文宏保护医生独立写病历、独立诊断的权利，实际上是在保护
人类医学数据的多样性和真实性
。
05
AI永远无法理解的幻觉与责任
还有一个老生常谈但致命的问题：
幻觉
（Hallucination）。
现在的LLM本质上是一个概率模型。它不是在
理解
医学，它是在做
文字接龙
。它预测下一个字出现概率最高的词是什么。
大部分时候，概率是对的。
但在医疗领域，那0.01%的错误概率，就是
医疗事故
。
AI会一本正经地胡说八道。它会编造一个不存在的药物剂量，它会把两种相互冲突的治疗方案混在一起，而且语气极其自信。
一个资深的医生，像张文宏这样的，一眼就能看出AI在瞎扯。
但是一个从实习期就被AI喂养长大的医生，他没有能力去质疑AI。
在心理学上，这叫
自动化偏差
（Automation Bias）。当人类面对一个看似权威的自动化系统时，会倾向于抑制自己的怀疑，盲目相信系统的判断。
尤其是当在这个系统这一个月来表现都不错的时候，医生的警惕性会降到最低。
然后，就在那个疲惫的夜班，AI给出了一个荒谬但看似合理的诊断，医生习惯性地点击了确认。
谁来负责？
算法工程师吗？别开玩笑了。我们在用户协议里写得清清楚楚：
本服务仅供参考，不构成医疗建议。
最终扛下所有的，还是那个医生，以及躺在病床上的患者。
把性命攸关的决策权，交给一个
无法承担法律责任、且存在不可控幻觉
的黑盒模型，这在伦理上就是走不通的。
06
我们需要的是副驾驶，不是自动驾驶
所以，张文宏反对的不是技术，他反对的是
技术对人的异化
。
他提倡的模式，其实就是
Human-in-the-loop
（人在回路）。
顺序至关重要。
正确姿势：
 医生看病人 -> 医生独立思考 -> 医生得出初步结论 -> 询问AI是否有遗漏 -> 医生综合判断 -> 形成病历。
错误姿势：
 医生看病人 -> 输入AI -> AI生成结论 -> 医生签字。
这一前一后的差别，就是
大师
与
操作员
的差别，就是
人类主宰工具
与
工具奴役人类
的差别。
我们不能因为有了计算器，就不教孩子背九九乘法表；也不能因为有了AI，就允许医生放弃诊断思维的训练。
在这个算法接管一切的时代，
保持大脑的痛感
，居然成了一种奢侈的修行。
张文宏教授的担忧，极其合理，且充满远见。
医疗行业是人类最后的防线之一。如果连这里的医生都变成了只会调包的
API调用师
，那我们离《瓦力》电影里那个四肢退化、躺在椅子上只会喝饮料的人类结局，真的就不远了。
拒绝被AI饲养，从坚持低效但独立的思考开始。
这一票，我投给张文宏。
