# 有哪些让你相见恨晚的记笔记方法？

- 日期: 2024-11-10
- 赞同: 0
- 评论: 0
- 链接: https://www.zhihu.com/question/324766011/answer/27564617776

---

受到拉维康特的影响，我之前做笔记的习惯是简化。我习惯于把复杂的观点简化成一两句话，甚至一两个词，作为知识管理的索引目录。这样的好处是条理清晰，容易梳理，方便画逻辑树。坏处是印象模糊。你能迅速联想到想到一个观点，但是总是说不清楚。 
读书会的活动暴露了我的这个问题。读书会考验的是“如何用一小段话把一个观点解释清楚”的能力，而不是一两个金句让大家猜，更不是说一大堆让人摸不着头脑。详略得当是聊天的最高境界。 
我现在做笔记的原则是：一个观点总结成一段话。一句话是谜语，几段话是啰嗦，一段话不多不少刚刚好。笔记必须是一段完整的话，而不是只言片语，否则复习的时候还是要靠猜，还要去读原文。笔记更不是长篇大论，否则还要做笔记的笔记。笔记必须拿来就能用。 
... 
这让我产生一个洞见：大脑并不擅长记忆单独的符号，但擅长记忆一个意群，一个模式，一种感觉。高效的笔记方式必须遵循大脑的最自然的工作模式，而大脑的工作模式是模块化的。 
读赫伯特西蒙的认知，说大脑之所以喜欢把信息模块化，是因为大脑同时只能处理四个左右的『信息模块』。一个信息模块里容纳的信息越多，模块化程度越高，大脑运算也就越高效。 
把关键信息模块化，方便重复利用，能极大的节约记忆空间。就像积木一样，你不可能拥有所有玩具，但只要有一套乐高积木，就能搭建出无限的可能。模块化是对信息的压缩，让你用有限的容量记住更多信息。 
不过这也带来了大脑的一个bug。由于关键信息模块都是重复使用的，所以你经常会有「似曾相识的感觉」。比如你在寒冷的冬天见了个朋友，结果你以后每次见到相似的朋友，都会感觉有一点冷。 
在聊天、写作和笔记中善用模块化思维，能极大的提升你的记忆效率。一个人可能聊起来滔滔不绝，或者著作等身，令人生畏，但仔细观察，他只是把相同的信息模块来回排列组合而已。高手能拨开迷雾，辩识出那些复杂花样下重复出现的基本模块，高手的世界是模块化的。 
模块化意味着大量的前期准备，也就是基本功。我们惊叹于记忆术表演者远超一般人记忆容量，但这其实是因为他们做了大量的前期准备工作。普通人要记忆remarkable这个单次，要记住十个字母，但如果你已经对英语词根很熟悉，按照remark-able来记，那就只需要记两个模块了。那些记忆术表演者具有惊人的记忆能力，背后其实都是这个道理。 
... 
把模块展开的能力也很重要。
读李敖回忆录，说自己之所以博闻强记，是因为他会刻意训练自己的思维：看见具体的东西，就把它变的抽象，看到抽象的的东西，就把它变的具体。概括和发散是最好的思维训练。 
概括就是形成信息模块的能力。张佳玮说，善于写东西的人都有强大的归纳能力，善于在零散的现象中发现规律。 
发散就是将信息模块展开的能力。我常说聊天和写作能锻炼记忆力，其实就是信息模块在不同层级上的展开。聊天是在一段话上的展开，写作是在一篇文章上的展开。而倾听时常要用一句话或一个词来回应对方，也就是在一个词上的展开。 
写作的高手用一篇文章把一个观点讲清楚。聊天的高手用一段话把一个观点讲清楚。倾听的高手用一个词把一个观点讲清楚。 
... 
顺嘴提一句， 
软件发展史就是底层代码不断组成固定模块，小模块又组成大模块的过程。 
早期的软件开发都是一对一定制，每个硬件的代码都是从头开始写的，相互间不能通用。聪明的程序员发现自己总是在重复相同的工作，于是就把重复的代码固定下来，可以在不同的硬件平台上重复使用，这样就不用再重复造轮子了。后来代码封装的层级越来越高，能够重复使用的模块越来越大，我们编程也就越来越方便。从刚开始的汇编，C，到现代的各种高级语言，协议、驱动、引擎，软件，本质上都是可重复使用的、为提高效率而诞生的代码模块。
